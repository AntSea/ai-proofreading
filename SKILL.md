# AI味去除（文章降重）

系统化降低文章AI检测率，让文章读起来像真人写的专业技能。

## 何时使用此Skill

当检测到以下场景时，自动加载此skill：
- 用户说"去除AI味"、"降重"、"太AI了"
- 用户说"像机器人写的"、"没人味"
- 用户提到"AI检测率高"、"想让它自然一点"
- 初稿创作完成后需要优化
- 用户分享AI生成的文章需要修改

## 核心目标

- **降低AI检测率**：目标控制在30%以下
- **增加人味**：让文章有观点、有态度、像真人在说话
- **保持可读性**：不损失信息量，让读者能看懂

## 输出格式

完成降重后，输出：
1. **修改后的文章** - 完整正文
2. **主要修改点** - 列出3-5个关键改动
3. **（可选）AI检测率** - 如用户要求，可提供预估变化

## 使用模式选择

### 快速模式（3分钟）
适用于：紧急修改、内容较短、日常微调
- 只做核心检查
- 重点：去掉最明显的AI腔

### 完整模式（15分钟）
适用于：重要文章、内容较长、需要深度优化
- 三遍完整审校
- 重点：全面降重+风格优化

**询问用户选择模式**：请告诉我你想用"快速模式"还是"完整模式"？

## 第二遍：AI腔识别与改写（核心环节）

### 6大类AI腔（经典版）

#### 1️⃣ 套话连篇
❌ **AI常说**：
- "在当今时代"、"在...的大背景下"
- "综上所述"、"总而言之"
- "值得注意的是"、"需要强调的是"
- "随着...的发展"、"伴随着..."

✅ **改写方向**：直接切主题，不绕弯子

**示例**：
```
❌ "在当今AI技术飞速发展的时代，编程工具也在不断进化..."
✅ "Claude Code出了。我用了两周，确实比Cursor好用。"
```

#### 2️⃣ AI句式（高频重复模板）
❌ **AI常说**：
- "不是...而是..."（连续出现3次以上）
- "不仅...而且..."（堆叠使用）
- "一方面...另一方面..."
- "既...又..."（过度使用）

✅ **改写方向**：拆成短句，用多样化表达

**示例**：
```
❌ "Claude Code不是简单的工具，而是智能助手。它不仅能写代码，而且能理解意图。"
✅ "Claude Code不只是补全代码。它能理解你的意图，知道你想做什么。"
```

#### 3️⃣ 书面词汇（距离感强）
❌ **AI常说**：
- "显著提升"、"大幅改善"→ 用具体数字
- "充分利用"→ "用好"
- "进行操作"→ 直接用动词（"点击"、"输入"）
- "实现功能"→ "做到"
- "促进发展"→ "帮助..."

✅ **改写方向**：口语化，具体化

**示例**：
```
❌ "通过充分利用Claude Code的能力，可以显著提升开发效率..."
✅ "用好Claude Code，你的开发速度能快不少。我测下来，平均每个项目省3-5小时。"
```

#### 4️⃣ 结构机械（AI式列表癖）
❌ **AI特征**：
- 过度使用"首先、其次、最后"
- 无意义的小标题密集（每200字一个）
- 能用散文讲清楚的，非要列表
- "以下几点"、"主要体现在以下方面"

✅ **改写方向**：自然叙事，列表适度

**示例**：
```
❌ "Claude Code的优势主要体现在以下几个方面：首先，它具有强大的上下文理解能力；其次，它支持多种编程语言；最后，它的响应速度很快。"

✅ "Claude Code最强的地方是什么？理解上下文。你让它改一个功能，它知道要动哪几个文件，不会漏掉。"
```

#### 5️⃣ 态度中立（无立场无观点）
❌ **AI特征**：
- 永远客观平衡："既有优点也有缺点"
- 不敢下判断："具体取决于实际情况"
- 每句话都加限定词："可能"、"或许"、"在某些情况下"

✅ **改写方向**：明确态度，敢下判断

**示例**：
```
❌ "Claude Code在某些场景下可能表现较好，但也存在一些局限性，具体效果取决于实际使用情况。"

✅ "Claude Code确实好用。处理大型项目时，比Cursor稳定多了。但价格贵，个人开发者可能扛不住。"
```

#### 6️⃣ 细节缺失（抽象不落地）
❌ **AI特征**：
- 大词多，具体细节少
- "许多"、"一些"、"显著"、"大量"（不给数字）
- 缺少真实感受和个人经历
- 无时间线、无场景、无冲突

✅ **改写方向**：加入真实细节、具体数字、个人经历

**示例**：
```
❌ "经过一段时间的使用，我发现Claude Code能够有效提升开发效率。"

✅ "用了两周。最明显的变化是，之前要3天的项目，现在1天半就能搞定。特别是重构老代码，Claude Code能一次性找出所有关联文件。"
```

---

### 4类新增AI腔（2025版）

#### 7️⃣ 车轱辘话（反复表达）
❌ **AI特征**：
- 同一意思用不同说法说三遍
- "换句话说"、"也就是说"、"即"
- 开头总结，结尾又总结，内容一样

✅ **改写方向**：删除重复，保留核心观点

**示例**：
```
❌ "Claude Code是一款AI编程工具，它能够理解代码上下文，帮助开发者编写代码。换句话说，它是一个智能编程助手，可以提高编程效率。也就是说，使用它可以让开发更快。"

✅ "Claude Code是AI编程工具，能理解代码上下文，帮写代码。"
```

#### 8️⃣ 正确但无用（说了等于没说）
❌ **AI特征**：
- 正确的废话："要具体情况具体分析"
- 正确的废话："需要根据实际情况选择"
- 正确的废话："建议多方比较再做决定"

✅ **改写方向**：给出具体建议或立场

**示例**：
```
❌ "选择编程工具需要根据实际情况具体分析，建议多方比较后再做决定。"

✅ "如果你主要写前端，选Cursor；如果你主要写后端，Claude Code更合适。"
```

#### 9️⃣ 过度总结（每段必总结）
❌ **AI特征**：
- 每个段落最后一句都是小结
- "综上所述"、"总而言之"高频出现
- 第一段是全文概述，最后一段又是概述

✅ **改写方向**：只在关键位置总结，避免重复

**示例**：
```
❌ "第一点，Claude Code上下文理解强。第二点，Claude Code支持多语言。第三点，Claude Code响应速度快。总结一下，Claude Code这三个优点都很突出。"

✅ "Claude Code最强的是上下文理解。改一个功能，它知道要动哪几个文件。速度也快，支持20多种语言。"
```

#### 🔟 EMOJI/格式滥用
❌ **AI特征**：
- 列表项前后都加emoji
- 每个句子都加emoji
- 格式过度：加粗、斜体、emoji堆砌
- 标题emoji化："🔥AI工具推荐🔥"

✅ **改写方向**：适度使用，每段1-2个重点即可

**示例**：
```
❌ "🔥AI工具推荐🔥
1️⃣ Claude Code
2️⃣ Cursor
3️⃣ Copilot

优点：
✅ 功能强大
✅ 智能
✅ 高效"

✅ "推荐几个AI编程工具：Claude Code、Cursor、Copilot。各有侧重，看你需求。"
```

- 专业感要有
- 可以稍长
- 逻辑要清晰
- 避免口语化过度

### 微博/即刻
- 更短
- 更随意
- 可以有观点和态度

## 常用改写技巧（快速记忆）

1. **长句拆短句**：超过30字，拆成2-3句
2. **抽象改具体**： "很多"→"3个"，"显著提升"→"省3-5小时"
3. **书面改口语**："进行操作"→"点击"
4. **删除车轱辘**：同一意思只说一遍
5. **态度明确化**：去掉"可能"、"或许"，改成"我觉得"
6. **细节真实化**：加入个人经历、具体数字

## 常见错误与改写示例

### 错误1：开头绕圈子
```
❌ "在当今AI技术飞速发展的背景下，编程工具领域也迎来了新的变革..."
✅ "Claude Code出了。我用了两周，确实比Cursor好用。"
```

### 错误2：过度使用AI句式
```
❌ "Claude Code不是简单的代码补全工具，而是一个智能编程助手。它不仅能够生成代码，而且能够理解开发者的意图。"
✅ "Claude Code不只是补全代码。它能理解你想干嘛。"
```

### 错误3：缺少具体细节
```
❌ "经过一段时间的测试，我发现Claude Code在很多方面都表现优秀。"
✅ "用了两周。之前要3天的项目，现在1天半就能搞定。"
```

### 错误4：态度过于中立
```
❌ "Claude Code有其优势，也存在一些不足，具体是否适合使用需要根据实际情况判断。"
✅ "Claude Code确实好用。但价格贵，个人开发者可能扛不住。"
```
